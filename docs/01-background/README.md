# 背景与场景

## 场景描述
运维人员希望通过一个 **CLI Agent** 以自然语言/结构化参数驱动“一键安装”，完成 YashanDB（及相关组件）在目标主机上的自动化安装与初始化。

现有安装工具已具备成熟能力：本仓库的 `yinstall` 会通过 SSH 在目标机上执行 OS 基线、数据库安装、主备扩容、YCM/YMP 安装、以及清理。

## 关键痛点
- 参数复杂、步骤多，容易手工执行出错
- 高危操作（清理/强制执行/删除用户组）需要严格门禁
- 安装过程需要完整日志与可回放记录，便于审计与复盘

## 本项目要做的事
- 在 `yinstall` 之上增加一层 **Agent Orchestrator**：
  - 把 `yinstall` 命令封装成“工具（Tool）”，提供强类型输入/输出
  - 将安装/清理过程固化成“技能（Skill / Workflow）”状态机
  - 提供交互确认、审批 token、以及审计记录

## 术语
- **Agent**：负责理解用户意图、选择工具、编排工作流的系统（可用本地模型或云端大模型）
- **Tool**：一个可调用的受控函数，映射到 `yinstall` 的某个子命令或能力
- **Workflow/Skill**：由多个 Tool 组成的可执行 SOP（例如“一键安装单机 DB”）
- **Target Host**：被安装的目标主机（`--targets`）
- **Control Plane**：运行 Agent/CLI 的本机

## 边界与假设（Design 阶段约束）
- 凭据由调用方提供（交互输入/环境变量/外部密钥管理），Agent 不长期存储明文密码
- 默认只支持 SSH 方式到目标机（与 `yinstall` 一致）
- MVP 优先支持：Linux 目标机 + 单机 DB 安装
